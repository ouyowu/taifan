## Fast daily flow

### 1. Preview

In the source board:

- click `先试抓`
- check:
  - is the title clear?
  - is the link correct?
  - does the summary look useful?
  - did this already enter the system?

If the preview looks weak, stop there.

### 2. Generate candidates

Only generate a small batch:

- `2 to 4` items is enough for a normal pass

Use:

- `用这条内容生成候选`

### 3. Review

Go to content review:

- `只看待发布`
- `只看自动整理`

Check:

- title clarity
- category correctness
- related star correctness
- whether the Chinese summary reads naturally

### 4. Publish

Recommended publish amount for a careful pass:

- `2 to 5` items

That is enough to keep the newsroom alive without flooding it.

## Good publish signals

Good candidates usually have:

- a clear artist name
- a clear recent update
- a readable title
- a summary that already sounds like a useful quick-read

## Bad publish signals

Pause or skip if:

- title is vague
- summary is weak
- category feels wrong
- related star is wrong
- source looks noisy
- candidate feels duplicated

## Daily success condition

A daily run is already good if:

- the main four sources preview correctly
- `2 to 4` good candidates are generated
- `2 to 5` items are published cleanly
- `/news` looks fresh and coherent
